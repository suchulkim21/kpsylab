# 배포 체크리스트

배포 전에 다음 항목들을 확인하세요.

## ✅ 사전 준비

### 코드 품질
- [ ] `npm run lint` 통과
- [ ] `npm run build` 성공
- [ ] `npm start` 로컬 테스트 성공
- [ ] 주요 기능 로컬에서 테스트 완료

### 환경 변수
- [ ] `NEXT_PUBLIC_SUPABASE_URL` 설정
- [ ] `NEXT_PUBLIC_SUPABASE_ANON_KEY` 설정
- [ ] `ADMIN_SECRET` 설정 (강력한 값)
- [ ] `SITE_URL` 설정
- [ ] (선택) `NEXT_PUBLIC_SENTRY_DSN` 설정

### 데이터베이스
- [ ] Supabase 프로젝트 활성화 확인
- [ ] RLS 정책 확인
- [ ] 필요한 테이블 생성 확인
- [ ] 마이그레이션 적용 확인

## 🚀 배포 단계

### 1. Git 커밋
- [ ] 모든 변경사항 커밋
- [ ] 의미 있는 커밋 메시지 작성
- [ ] `main` 브랜치에 푸시

### 2. Vercel 설정
- [ ] 프로젝트 연결 확인
- [ ] 환경 변수 모두 설정
- [ ] 빌드 설정 확인 (Root: `apps/portal`)

### 3. 배포 실행
- [ ] Vercel CLI 또는 대시보드에서 배포
- [ ] 빌드 로그 확인
- [ ] 배포 성공 확인
