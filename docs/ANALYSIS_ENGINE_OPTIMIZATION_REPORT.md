## 1. 개요

### 1.1 배경

성장 로드맵 시스템의 질문 선택지가 기존 **2-3개**에서 **4개**로 통일되었습니다. 이에 따라 분석 엔진의 최적화 작업이 수행되었으며, 본 문서는 해당 변경이 분석 결과의 정확도에 미치는 영향을 상세히 분석합니다.

| 구분 | 파일 | 변경 유형 |
|------|------|----------|

### 3.2 분석 엔진 (`analysis.ts`)

#### 3.2.1 Percentile 계산 함수 분리

```typescript
// 이전: 인라인 계산

// 현재: 함수로 분리 및 임계값 조정
export function calculatePercentile(percent: number): string {
    if (percent >= ANALYSIS_CONSTANTS.HIGH_PERCENTILE_THRESHOLD) {  // 55%

    } else if (percent >= ANALYSIS_CONSTANTS.MID_PERCENTILE_THRESHOLD) {  // 35%

    } else {
        return "하위 30%";
    }
}
```

## 4. 분석 결과 영향 분석

### 4.1 선택지 증가의 영향

#### 4.1.1 응답 분포 변화

**이전 시스템 (3개 선택지)**:
```
응답자의 선택 확률 분포:
- A (acting):   ~33%
- B (thinking): ~33%
- C (feeling):  ~33%
```

**현재 시스템 (4개 선택지)**:
```
응답자의 선택 확률 분포:
- A (acting):   ~25%
- B (thinking): ~25%
- C (feeling):  ~25%
- D (협력/관찰): ~25%
```

#### 4.1.2 점수 분산 변화

4개 선택지 환경에서는 점수 분포가 더 넓어질 수 있습니다:

```
이전 (3개 선택지, 27개 시나리오):
- 가능한 점수 조합: 3^27 = 약 7.6조 개
- 주도성(P) 범위: 약 -135 ~ +270
- 적응성(A) 범위: 약 -135 ~ +405  
- 사회적 거리(SD) 범위: 약 -270 ~ +270

현재 (4개 선택지, 27개 시나리오):
- 가능한 점수 조합: 4^27 = 약 1.8경 개
- 점수 범위: D 선택지 가중치에 따라 변동
```

### 4.2 D 선택지 가중치 패턴 (v2.1 Detached Observer 재설계)

D 선택지는 **Detached Observer(냉철한 관찰자)** / **Independent Strategist(독자적 전략가)** 페르소나로 재설계됨:

| 시나리오 유형 | D 선택지 특성 | 가중치 패턴 |
|--------------|--------------|------------|
| 자극 통제 (Theme 1) | 개입 포기, 관찰, 책임 회피 | P: -10~0, A: -8~0, SD: 10~15 |
| 강화 패턴 붕괴 (Theme 2) | 몰입 포기, 전략적 후퇴, 손절 | P: -10~5, A: -5~5, SD: 10~12 |
| 사회적 아키타입 (Theme 3) | 책임 한정, 방관, 연대 거부 | P: -8~5, A: -5~0, SD: 10~15 |

**특징** (명확한 기회비용 설계):
- D 선택지는 **높은 사회적 거리(socialDistance)** + **낮은/음수 주도성·적응성**
- 개입 포기, 관계 단절, 자기보호 등 구체적 행동 묘사
- 감정에 휘둘리지 않되 조직에 헌신하지 않는 '제3의 태도'

### 4.3 동일 응답자의 결과 변화 시나리오

#### 시나리오 1: 기존 A, B, C만 선택하는 경우

```
결과: 이전 시스템과 동일한 결과
영향: 없음
```

#### 시나리오 2: D 선택지를 일부 선택하는 경우

```
예시: 27개 시나리오 중 9개에서 D 선택

이전 결과 (가정):
- proactivity: 120
- adaptability: 150
- socialDistance: 80

현재 결과 (D 선택 반영):
- proactivity: 105 (-15)
- adaptability: 180 (+30)
- socialDistance: 50 (-30)

영향: 
- 지배적 특성(dominant trait)이 바뀔 수 있음
- Percentile 해석이 달라질 수 있음
```

## 5. 정확도 비교 분석

### 5.1 정확도에 영향을 미치는 요소

| 요소 | 영향도 | 설명 |
|------|--------|------|
| 선택지 수 증가 | ⚠️ 중간 | 더 세분화된 응답 가능, 하지만 기존과 다른 결과 |
| D 선택지 가중치 | ⚠️ 중간 | 새로운 행동 패턴 포착, 점수 분포 변화 |
| Percentile 임계값 변경 | ⚠️ 중간 | 같은 점수도 다른 해석 가능 |
| 일관성 검사 임계값 변경 | 🟢 낮음 | 신뢰도 점수 약간 변동 |

### 5.2 정확도 유형별 분석

#### 5.2.1 내적 일관성 (Internal Consistency)

```
이전 시스템: 3개 선택지로 제한된 응답 패턴
현재 시스템: 4개 선택지로 더 다양한 응답 패턴 허용

평가: 
- 이론적으로 내적 일관성이 더 높아질 수 있음
- 단, 검증 데이터가 필요함
```

#### 5.2.2 예측 타당도 (Predictive Validity)

```
이전 시스템: 기존 가중치 기반 예측
현재 시스템: D 선택지 가중치 추가로 예측 모델 변화

평가:
- D 선택지 가중치가 실제 행동과 일치하는지 검증 필요
- 현재 D 가중치는 이론적 설계에 기반 (실증 데이터 없음)
```

#### 5.2.3 구성 타당도 (Construct Validity)

```
이전 시스템: 3개 차원 (P, A, SD)
현재 시스템: 동일한 3개 차원, D 선택지는 기존 차원에 매핑

평가:
- 구성 타당도는 유지됨
- D 선택지가 기존 구성을 더 풍부하게 측정
```

### 5.3 Percentile 임계값 변경의 영향

#### 상세 비교

```
예시: 응답자 점수 비율이 52%인 경우

이전 시스템 (임계값 60/40%):
- 52% → "중위 30-70%" (40% ≤ 52% < 60%)

현재 시스템 (임계값 55/35%):
- 52% → "중위 30-70%" (35% ≤ 52% < 55%)

→ 이 경우 동일한 결과

예시: 응답자 점수 비율이 57%인 경우

이전 시스템:
- 57% → "중위 30-70%" (40% ≤ 57% < 60%)

현재 시스템:
- 57% → "상위 30%" (57% ≥ 55%)

→ 해석이 달라짐!
```

## 8. 결론

### 8.1 요약

1. **변경된 것**: 선택지 수 (2-3개 → 4개), 분석 엔진 임계값

4. **위험**: 결과 불일치, 사용자 혼란

### 8.2 최종 권장

| 항목 | 권장 사항 |
|------|----------|
| 임계값 | 기존 값 유지 또는 원복 (의사결정 필요) |

### 9.3 정리

- **MNPS**: 리포트가 "점수 중심"이 아니라 **맥락·패턴·해석** 중심으로 읽히도록 문구 정리.
- **성장 로드맵**: 결과 화면의 **전략 요약·실행 권고·성장 방안**이 전략/차원에 따라 구체적인 문장으로 표시되도록 콘텐츠 모듈 및 엔진 임계값 정리 완료.
