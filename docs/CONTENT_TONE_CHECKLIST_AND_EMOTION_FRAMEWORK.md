## 1. 블로그·결과 톤 체크리스트

발행 전(블로그 포스트, 결과 페이지 문구, 가이드·FAQ 문장)에 아래를 1인이라도 한 번 확인하는 것을 권장합니다.

### 1.1 철학·맥락

| 체크 항목 | 예시/기준 |
|-----------|-----------|
| **검증된 심리학·뇌과학 맥락**이 있는가? | 주장에 “어떤 연구·이론·윤리 기준”과 연결되는지 한 줄이라도 드러남. |
| **이분법적 판단을 강화하지 않는가?** | “정상/비정상”, “선/악” 이분법보다 구간·맥락·상황 의존 표현 사용. |
| **관계·질서가 드러나는가?** | 정보 나열이 아니라 “이것과 저것의 **관계**”, “**순서·질서**가 만드는 의미”가 문장/구성에 반영됨. |

### 1.2 톤·품격

| 체크 항목 | 예시/기준 |
|-----------|-----------|
| **가벼운 블로그 톤을 피하는가?** | “묵직한 통찰”, “배경 설명”, “객관적 관점”을 담고 있는지. |
| **사과·불필요한 미사여구**를 줄였는가? | 정확한 정보와 통찰로 신뢰를 구축하는 문장인지. |
| **사용자 가치관 추측** 문구가 없는가? | “당신은 ~일 거예요”보다 “이 맥락에서는 ~로 해석될 수 있습니다” 등. |

### 1.3 결과·리포트 전용

| 체크 항목 | 예시/기준 |
|-----------|-----------|
| **“고정된 정답”이 아닌 “맥락에서의 한 도면”**으로 읽히는가? | “현재 응답 맥락”, “시점·상황이 바뀌면 다르게 읽힐 수 있음” 등 노출. |
| **Good/Bad가 “선/악”이 아닌 “두 관점”**으로 프레이밍되는가? | 엘리트 관점 vs 리스크·어두운 이면 관점으로 설명. |
| **진단이 아님**이 명시되는가? | 임상·진단 대체 불가, 참고용·자기 성찰용 문구 포함. |

## 2. 감정 프레임워크 요약 (CONTENT_BACKLOG 기반)

“일반적인 사람들이 느끼는 감정” 6가지 카테고리:

| 카테고리 | 설명 |
|----------|------|
| **불안·걱정** | 미래, 관계, 평가에 대한 |
| **외로움** | 연결이 끊어졌을 때 |
| **무의미감** | 목표나 의미를 찾기 어려울 때 |
| **기쁨·안도** | 소소한 일에도, 큰 일에도 |
| **답답함** | 말이나 상황이 막혀 있을 때 |
| **지침·피로** | 에너지와 의미가 닳아갈 때 |

## 3. 감정 프레임워크 활용 계획

### 3.1 블로그 시리즈 주제

- **카테고리별 시리즈**: 위 6가지 중 하나를 테마로 한 포스트 시리즈 기획.
  - 예: “불안·걱정” 시리즈 — 미래 불안, 관계 불안, 평가 불안을 각각 심리학·뇌과학 맥락으로 다룸.
- **톤**: 독자가 그 감정을 “이상한 것”이 아니라 **맥락 속에서 이해**할 수 있도록, 비이분법·연결성 원칙 적용.

### 3.2 MNPS·성장 로드맵 결과 해석에 감정 맥락 반영

- **결과 문구 설계 시**: “이 프로파일을 가진 사람이 **어떤 상황에서 어떤 감정**을 더 느끼기 쉬운가”를 선택적으로 문장에 녹임.
  - 예: 높은 불안 맥락에서 테스트한 경우 vs 안도·기쁨 맥락에서의 해석 차이를 “이번 응답 맥락”과 연결해 언급.
- **리포트 블록**: “지침·피로”, “답답함” 등이 독자 상태와 맞닿는 문장을 넣을 때, 감정 프레임워크 용어를 참고해 **일관된 표현** 사용.

### 3.3 콘텐츠 톤앤매너 가이드 (독자 감정에 맞춘 표현)

- **독자가 느끼는 감정**에 맞춘 표현:
  - 불안·걱정 → “맥락을 같이 보면”, “단계를 나누어 보면” 등 **구조·예측 가능성**을 주는 문구.
  - 외로움 → “연결”, “관계의 질서” 등 **연결성** 원칙과 맞는 메시지.
  - 무의미감 → “이 도면이 다른 선택과 어떻게 연결되는지” 등 **질서·다음 단계** 제시.
  - 기쁨·안도 → 과장 없이 “이 해석이 주는 한 가지 관점”으로 마무리.
  - 답답함 → “한 번에 정리되지 않는 게 맞다”, “회색지대” 등 **비이분법** 강화.
  - 지침·피로 → 문장 길이·블록 수 조절, “한 번에 하나씩” 등 **인지 부하** 완화.

### 3.4 우선순위 제안

| 순서 | 활용처 | 내용 |
|------|--------|------|
| 1 | **블로그 시리즈** | 6감정 중 1~2개 카테고리로 먼저 시리즈 기획·1차 포스트 작성. |
| 2 | **결과 해석 문구** | MNPS Good/Bad, 성장 로드맵 요약 문장에 “감정 맥락” 1~2문장 선택 반영. |
| 3 | **톤 가이드** | 위 “독자 감정별 표현”을 CONTENT_TONE_GUIDE 또는 본 문서에 상세화해 팀·AI 생성 시 참조. |
