# MNPS 콘텐츠 전략: 문제 은행과 분석 리포트 중심

**문서 성격**: MNPS 서비스에서 **콘텐츠의 위치**와 **재검사 대응**을 정의하는 전략 문서.  
**참조**: PROJECT_GAPS_AND_REMEDIATION §5, MNPS_TEST_OVERVIEW, MNPS_ARCHETYPE_SYSTEM, **QUERY_RESPONSE_ANALYSIS_REPORT** (질의·응답 → 분석 레포트 파이프라인)

---

## 1. 서비스의 중심: 분석 리포트

- **문항·선택지는 콘텐츠의 시작(입력)일 뿐**이며, **최종 콘텐츠가 아니다.**
- **테스트 진행 후 사용자에게 보여줄 분석 리포트**를 작성·확장하는 것이 이 서비스의 **핵심**이다.
- Good/Bad 이중 리포트, 아키타입·시너지·딥다이브·D요인·Final Ruin 등은 모두 **분석 리포트(결과·해석)** 의 구성 요소이며, 이 부분의 품질과 분량이 사용자 가치를 결정한다.
- 따라서 "42문항 + 리포트 조합이 부족하지 않다"는 판단은 **틀린 전제**다. 문항 수는 재검사 경험과 함께 논의해야 하고, **리포트(분석 콘텐츠)** 가 서비스의 중심임을 명시해야 한다.

---

## 2. 재검사와 문제 은행(Item Bank)

- **2~3회 재검사하는 사용자**가 있다. 매번 **같은 42문항**을 반복하면, 같은 문답을 반복하는 것이 되어 **재검사의 의미가 없다.**
- 대응: **충분한 문항을 확보한 문제 은행(Item Bank)** 을 구축한다.
  - Dark Tetrad·D-Factor·검증·시나리오 각 **카테고리별**로, **동일 구인을 측정하는 대체 문항**을 여러 개 개발한다.
  - 회차별 또는 **랜덤 샘플링**으로 매 검사마다 42문항(또는 정한 N문항)을 은행에서 선출하여, 재검사 시에도 **다른 문항 조합**이 나오도록 한다.
- 문항 풀의 크기·구인당 최소 문항 수·채점 공식과의 호환(가중치·trait/subFactor 매핑)은 별도 설계 문서에서 정한다.

---

## 3. 콘텐츠 계층 정리

| 계층 | 역할 | 현재 상태 |
|------|------|-----------|
| **입력** | 문항·선택지 | **문제 은행 126문항**(42슬롯×3변형). 매 검사마다 슬롯당 1문항 랜덤 출제 → 42문항. 재검사 시 다른 조합. |
| **중간** | 채점·규준·분석 정확도 | 구현됨. |
| **최종(서비스 중심)** | **분석 리포트** (Good/Bad, 아키타입·시너지·딥다이브·D요인·Final Ruin, 규준 문구 등) | 구현됨. **이 부분의 확장·품질이 서비스의 핵심**임을 기획·개발에서 공유. |

---

## 4. 보완 우선순위

1. **전략 공유**: "분석 리포트가 서비스의 중심, 문항은 그 입력"을 PROJECT_GAPS_AND_REMEDIATION·MNPS 관련 문서에 반영. (반영됨)
2. **문제 은행**: 구현 완료. 126문항(42×3), 슬롯당 랜덤 1문항 → 42문항 출제. DB 마이그레이션 `007_mnps_question_bank_126.sql`, 채점은 baseId 기준 일관성·검증 문항 매칭.
3. **분석 리포트 확장**: Good/Bad·딥다이브·시너지·Final Ruin 등 **리포트 내 문단·블록**을 늘리고, 맥락·깊이를 높이는 콘텐츠 작업을 지속.
