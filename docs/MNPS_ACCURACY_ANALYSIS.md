# MNPS(Dark Nature Test) 분석 결과 정확도 분석

## 1. 요약

| 항목 | 상태 | 설명 |
|------|------|------|
| **채점 구조** | ⚠️ 부분적 | 36문항 중 **28문항만** 점수에 반영됨 |
| **임계값/규준** | ❌ 없음 | high/mid/low 구간(75, 45)이 **임의 설정**, 규준·타당도 검증 없음 |
| **해석 로직** | ⚠️ 규칙 기반 | 점수 구간별 **미리 작성된 문장** 조합, 개인별 검증 없음 |
| **방어/일관성 검사** | ❌ 미적용 | Defensive 플래그는 **현재 사용 중인 리포트(assembleReport)에 미포함** |
| **시나리오·검증 문항** | ❌ 미반영 | 시나리오 4문항, 검증 4문항이 **채점·해석에 전혀 사용되지 않음** |

---

## 2. 채점이 실제로 반영되는 문항

- 채점/해석에 쓰이는 답변은 **`trait` 또는 `subFactor`가 있는 문항만** 포함됩니다.
- **Validation(4문항)**: `trait`/`subFactor` 없음 → **점수·해석에 사용되지 않음**
- **Scenario(4문항)**: `trait`/`subFactor` 없음 → **점수·해석에 사용되지 않음**

따라서 **36문항 중 28문항만** trait/subFactor 점수 및 D점수·아키타입·리포트에 기여합니다.

```
포함: Dark Tetrad 16문항 (나르시시즘·마키아벨리즘·사이코패시·사디즘 각 4문항)
     + D-Factor 12문항 (이기주의·권리의식·도덕적이탈·악의성)
미포함: Validation 4문항, Scenario 4문항
```

---

## 3. 점수 계산 방식과 한계

### 3.1 Trait 점수 (0~100)

- 해당 trait가 붙은 문항의 **1~5 리커트 평균**을 (1→0, 5→100)으로 선형 변환.
- **문항 수**: 트레이트당 4문항으로 균형 있음.
- **한계**: 역문항은 일부만 적용(v3 등). 검증·시나리오 문항은 아예 반영 안 됨.

### 3.2 D-Total (종합 D 점수)

- `weightedSum / weightTotal * 100` + **Inter-correlation 가산** (4개 trait 평균 × 0.2, 최대 약 20점).
- 가중치: 트레이트/서브팩터별로 0.9~1.1 사용.
- **한계**:
  - 가중치와 가산 공식이 **문헌/표준화된 척도와 일치하는지 검증되지 않음**.
  - 규준(평균, 표준편차, 백분위)이 없어 **“얼마나 높은/낮은지”를 객관적으로 말할 수 없음**.

### 3.3 아키타입·해석 구간

- **high**: 75 이상, **mid**: 45~75 미만, **low**: 45 미만.
- **한계**: 75/45는 **임의 설정**이며, 표준화된 Dark Tetrad 척도나 규준 연구와 연결되어 있지 않음.

---

## 4. 해석(리포트) 정확도

### 4.1 Good/Bad 리포트 생성 방식

- **assembleReport**:
  - 4개 trait 점수만 사용 (subFactor 점수는 **아키타입·스니펫 선택에 사용 안 함**).
  - 아키타입: 상위 2개 trait 조합 등 **규칙 기반**으로 결정.
  - 문장: **contentLibrary에 미리 작성된 스니펫**을 점수 구간(high/mid/low)에 따라 조합.

즉, **“측정된 점수 → 규칙으로 구간 분류 → 해당 구간에 매핑된 문장 노출”** 구조입니다.

### 4.2 정확도 관점의 한계

1. **규준 부재**  
   - “high/mid/low”가 실제 분포에서 어떤 백분위에 해당하는지 알 수 없음.  
   - 따라서 “당신은 전략적 직관이 높다” 같은 해석이 **통계적으로 어떤 수준을 의미하는지 불명확**합니다.

2. **타당도 미검증**  
   - 문항이 Dark Tetrad/D-Factor를 실제로 측정하는지(**구인 타당도**) 검증되지 않음.  
   - 다른 표준 척도(SD3, SRP 등)와의 **동시 타당도**도 없음.

3. **신뢰도 미검증**  
   - 동일인 재측정 시 점수가 얼마나 비슷한지(**재검 신뢰도**), 문항 간 일관성(**내적 일관성**) 등이 보고되지 않음.

4. **방어/일관성 검사 미사용**  
   - `calculateDarkProfile`에는 “spread < 15 && totalDScore >= 60”일 때 **Defensive/Manipulative** 문구가 있으나,  
   - **현재 결과 페이지가 사용하는 `assembleReport`에는 이 로직이 없음.**  
   - 따라서 “평탄하게 보정된 응답”을 의도한 검증 문항·시나리오 문항이 채점에도 쓰이지 않고, 방어 플래그도 사용되지 않습니다.

5. **시나리오·검증 문항 미활용**  
   - 시나리오 4문항: “복합 성향 측정”으로 설계되었으나 **trait/subFactor 매핑이 없어 채점/해석에 0% 반영**.  
   - 검증 4문항: “일관성·사회적 바람직성” 검출 목적이지만 **채점·플래그에 전혀 사용되지 않음**.

---

## 5. 정확도·신뢰도를 높이려면 (보완 제안)

| 우선순위 | 보완 내용 |
|----------|-----------|
| **1** | **Validation 문항 활용**: 검증 4문항 점수(또는 일관성 지표)를 채점/해석에 반영하고, `assembleReport`에 **Defensive/일관성 경고** 로직 추가. |
| **2** | **Scenario 문항 반영**: 시나리오 4문항에 trait/subFactor 매핑(또는 복합 규칙)을 정의하고, 채점·해석에 포함. |
| **3** | **규준·임계값 정당화**: 표준 Dark Tetrad/D-Factor 문헌 또는 자체 수집 데이터로 평균·표준편차·백분위를 산출하고, high/mid/low를 규준에 맞게 재설정. |
| **4** | **타당도·신뢰도 검증**: (선택) 표준 척도와의 상관, 재검 신뢰도, 내적 일관성(Cronbach α) 등을 보고하면 해석에 대한 신뢰가 높아짐. |
| **5** | **SubFactor 활용**: 아키타입·스니펫 선택에 subFactor 점수를 반영해 해석을 더 세분화. |

---

## 6. 결론

- **현재 시스템**은 “선택된 28문항의 자기보고 + 고정된 규칙 + 미리 쓴 문장”으로 **일관된 규칙 기반 결과**를 내는 수준입니다.
- **심리측정적 정확도(타당도·신뢰도·규준)** 는 아직 검증되지 않았고,  
  **검증·시나리오 8문항이 채점/해석에 쓰이지 않으며**,  
  **방어/일관성 검사도 현재 사용 중인 리포트(assembleReport)에 포함되어 있지 않습니다.**

따라서 “분석 결과의 정확도”에 대한 믿음을 갖기 위해서는,  
위 **보완 제안 1~3**(검증·시나리오 반영, 규준·임계값 정당화)을 우선 적용하고,  
가능하면 **4**(타당도·신뢰도 검증)를 추가하는 것이 좋습니다.
