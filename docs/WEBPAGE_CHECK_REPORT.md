**대상**: https://www.kpsylab.com  
**점검 기준**: 코드베이스·문서·플래그·접근성·콘텐츠 정합성

## 1. 발견된 문제와 조치

### 1.1 MNPS 문항 수 오류 (수정 완료)

| 항목 | 내용 |
|------|------|
| **문제** | 서비스 소개 페이지에서 MNPS를 "120문항 하이브리드 질문"으로 표기 |
| **실제** | MNPS는 **42문항** 구조 (MNPS_TEST_OVERVIEW, questions.ts 기준) |
| **조치** | `app/services/page.tsx` 문구를 "42문항 구조화 질문"으로 수정 |

### 1.2 /mnps 경로 404 (수정 완료)

| 항목 | 내용 |
|------|------|
| **문제** | 서비스 소개의 "MNPS 테스트 시작하기", 하단 CTA가 `href="/mnps"`로 연결되는데, `app/mnps/page.tsx`가 없어 **/mnps 접속 시 404** 발생 |
| **조치** | `app/mnps/page.tsx` 추가 — `/mnps` 접속 시 `/mnps/test`로 리다이렉트. 서비스 소개 내 링크는 `/mnps/test`로 통일 |

### 1.3 기능 플래그와 홈·서비스 소개 불일치 (수정 완료)

| 항목 | 내용 |
|------|------|
| **문제** | `MNPS_ENABLED`, `MIND_ARCHITECT_ENABLED`가 `false`인데, **홈**에서는 MNPS 카드가 플래그 없이 항상 노출되고, **서비스 소개**에서도 MNPS 블록이 항상 노출됨. 네비게이션만 플래그를 적용해 일관성 부족 |
| **조치** | ① 홈: MNPS 카드에 `MNPS_ENABLED` 적용. ② 서비스 소개: MNPS 섹션·하단 CTA에 `MNPS_ENABLED` 적용. ③ 두 플래그 모두 꺼져 있을 때 홈·서비스 소개에 "테스트 서비스는 준비 중입니다" 안내 노출 |

## 2. 현재 설정 상태 (참고)

| 플래그 | 기본값 | 영향 |
|--------|--------|------|
| `BLOG_ENABLED` | false | 네비에 "블로그" 미노출 |
| `MIND_ARCHITECT_ENABLED` | false | 네비·홈·서비스 소개에서 마인드 아키텍터 미노출 |
| `MNPS_ENABLED` | false | 네비·홈·서비스 소개에서 MNPS 미노출 |

**프로덕션에서 MNPS·마인드 아키텍터·블로그를 보이게 하려면** `lib/constants/featureFlags.ts`에서 해당 값을 `true`로 변경 후 배포하면 됩니다.

## 4. 라이브 사이트와의 정합성

검색 결과상 [kpsylab.com](https://www.kpsylab.com/)에는 "MNPS", "마인드 아키텍터", "블로그", "게시판", "관리자"가 노출된 것으로 보입니다.  
현재 코드에서는 **플래그가 모두 false**이면 네비에는 "서비스 소개", "게시판", "관리자"만 보이고, MNPS·마인드 아키텍터·블로그는 보이지 않습니다.  
배포 환경에서 플래그를 true로 둔 상태라면 라이브와 일치하며, false라면 위와 같이 "준비 중" 안내가 보이도록 수정된 상태입니다.
