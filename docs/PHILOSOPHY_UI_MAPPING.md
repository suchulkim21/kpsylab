## 1. 목적

새 모듈·기능·페이지를 설계할 때 **"이 기능이 맥락적 존재 철학을 UI적으로 어떻게 표현하는가?"**를 질문하고, 아래 매핑에 따라 일관된 경험을 제공한다.

## 2. 핵심 원칙 → UI 표현

### 2.1 가변성 (모든 존재와 가치는 맥락에 따라 변화한다)

| UI 요소 | 적용 방법 |
|---------|-----------|
| **결과·리포트** | 결과를 "고정된 정답"이 아닌 "**이 맥락에서의 한 해석**"으로 노출. 상단 또는 도입부에 "이 결과는 당신의 **현재 응답 맥락**에서 나온 한 장의 도면이며, 시점·상황이 바뀌면 다르게 읽힐 수 있습니다" 등 문구 포함. |
| **라벨·타이틀** | "당신은 A타입입니다"보다 "**현재 응답 기준으로** A에 가까운 프로파일이 관찰됩니다", "이 순간의 맥락에서" 등 **시점·조건**을 드러내는 표현 사용. |
| **재검사·비교** | 이전 결과와의 비교가 있으면 "**그때와 지금**의 맥락이 다르므로, 차이는 '변화'이자 '다른 각도에서의 도면'"으로 해석 가능함을 안내. |
| **버튼·CTA** | "결과 확인"보다 "**이번 응답의 도면 보기**", "맥락별 해석 더 보기" 등 **일회성·맥락 의존**을 암시하는 문구 검토. |

### 2.2 비이분법 (선/악, 정상/비정상의 경계는 유동적이다)

| UI 요소 | 적용 방법 |
|---------|-----------|
| **차트·시각화** | 점이나 구간을 "높음=나쁨, 낮음=좋음" 이분법으로만 보이지 않게. **연속축·구간·회색지대**를 표현하고, "이 구간은 특정 상황에서 강점이 될 수 있고, 다른 상황에서는 주의가 필요할 수 있습니다" 등 **상황 의존** 문구 보조. |
| **Good / Bad 리포트** | "Good=선, Bad=악"이 아니라 "**엘리트 관점의 해석**" vs "**리스크·어두운 이면의 해석**"으로 프레이밍. 두 관점이 **같은 데이터의 다른 각도**임을 명시. |
| **배지·상태** | "정상/비정상", "합격/불합격" 대신 "**상위 N% 구간**", "이 프로파일에서 주의할 수 있는 지점" 등 **구간·맥락** 기반 표현. |
| **폼·선택지** | "맞다/틀리다"가 아닌 "**그런 편이다 / 그렇지 않은 편이다**" 등 **정도·스펙트럼**을 강조하는 라벨. |

### 2.3 연결성 (정보 간 관계와 질서가 새로운 의미를 만든다)

| UI 요소 | 적용 방법 |
|---------|-----------|
| **결과 구성** | 단일 점수만 강조하지 않고 **특성 간 관계**(시너지, 프로파일), **이전 결과와의 연결**(재검사 시 변화), **블록화·연결** 메타포를 시각·문장으로 노출. |
| **내비게이션·플로우** | "테스트 → 결과"가 끊긴 단계가 아니라 "**이정표 → 다음 이정표**", "이 도면이 다른 서비스(성장 로드맵 등)와 어떻게 연결되는지" 안내. |
| **카드·섹션** | 파편화된 정보 나열보다 **"이 블록과 저 블록이 만드는 의미"**를 짧은 문장으로 연결. (예: "위 네 가지 특성의 **조합**이 당신의 프로파일을 이룹니다.") |
| **빈 상태·에러** | "데이터 없음"만 쓰지 않고 "**아직 이 맥락에서 그려진 도면이 없습니다**", "연결이 끊어졌습니다. 다시 시도하면 이어서 그릴 수 있습니다" 등 **관계·연속성**을 암시하는 문구. |

## 3. 새 기능 설계 시 체크리스트

- [ ] **가변성**: 이 화면의 결과/상태가 "고정된 정답"이 아니라 "맥락에 따른 한 해석"으로 읽히는가? 시점·조건을 드러내는 문구가 있는가?
- [ ] **비이분법**: 선/악, 정상/비정상 이분법을 강화하지 않는가? 구간·연속체·회색지대가 표현되는가?
- [ ] **연결성**: 단일 데이터가 아닌 **관계·질서·다음 단계**가 UI에 반영되는가? 블록화·연결 메타포가 활용되는가?
- [ ] **맥락 의존**: 같은 버튼/문구라도 **페이지·사용자 상태·이전 행동**에 따라 다르게 두는 것이 맞는가? (필요 시 분기)
